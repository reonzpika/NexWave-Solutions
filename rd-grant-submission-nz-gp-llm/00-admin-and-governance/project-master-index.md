## Version Control and File Naming Conventions

### Version Format
- **v1.0** = First approved version
- **v1.1** = Minor update (typo, clarification)
- **v2.0** = Major update (scope change, new section)

### File Naming Pattern
```
[Document-Name]_v[Major].[Minor]_[Status]_[YYYYMMDD].ext

Examples:
- DPIA_v1.0_Approved_20260131.pdf
- Cost-Template_v1.0_Draft_20260115.xlsx
- Transparency-Page_v1.2_Published_20260301.pdf
```

### Status Labels
- **Draft** = Work in progress
- **Review** = Awaiting approval
- **Approved** = Signed off
- **Published** = Live/external-facing

### Change Log Location
See `00-admin-and-governance/risk-and-change-management.md` for all document version history
